Bid Scores

Top  Previous  Next

 

To determine the best bid, Bridge Captain compares the bidder’s hand with the definitions on all possible bids.  The bid with the best score is selected.  A bid's score loses three points for each HCP or Total Point outside the indicated range. It loses one point for each card in a suit that is outside the parameters of the bid definition. Falling short of other criteria results in a score of  -200.

 

If a hand meets all the criteria for a definition in the Sequence File, then that definition is given a score of 200.

 

If a hand is TOO GOOD, e.g. has too many points, for the bid difinition, then the score will be negative, but will be printed in blue. If a hand is not good enough, then the negative score will be printed in red.

 

Sometimes the best bid in a given situation might actually have a negative score. When multiple bids tie for the best definition, then the highest ranking bid is chosen.**

 

When a bid has multiple definitions, the definition with the highest score is chosen for this bid.

 

**Please note the "R" (rank) exception under Other Bid Information.

 

Bidding Logic

Top  Previous  Next

 

The logic for bidding in Bridge Captain comes from two sources, in descending order of importance:

1. the Sequence File

2. the Logic Section

 

When you use the "Test Bid", "Hint", or "Explain Bid" features of the Bid Pad, you can tell the source of the Bid Definition by looking at the color of the word "Cnvt" in the Bid Definition Box.

 

White for the Sequence File

Yellow for the Logic Section.

 

It is possible for the user to change any bid's definition in the Sequence File, but not in the Logic Section.

 

If you see a bid that is defined poorly, you can correct it by placing a new or edited definition in the Sequence File, which will take precedence over any previous definition.

 

Bidding Score

 

The current bidder's hand is compared with the bid meaning shown in the BDB and a score is determined.

 

A hand that meets all the criteria for a meaning will have a score of 200 if the meaning comes from the Sequence File, and will have a lower, but posititve socre, if the meaning comes from Bridge Captain's coding.

 

A negative bidding score shows that the hand did not meet all the criteria. In general, a hand loses one "point" for each card less than, or more than, a suit's requirements, and one "point" for each HCP or Total Point outside the range indicated.

Editing a Bid's Description

Top  Previous  Next

 

Any bid's description may be edited, or added in the Bid Definition Box after pressing the Hint or Test Bid buttons. You can view the bid's description, but not edit it, after pressing Explain Bid button.

 

The Seq is determined from the bidding up to the current point in the auction and the Cnvt is the section of the Sequence File that holds this Bid Definition. If Cnvt is blank, it indicates that the bidding information was taken from the Logic Section, rather than from the Sequence File. Points, losing trick count (LTC), and information on each of the four suits are contained within unique text boxes. Other Bid Information is shown in a single text box at the bottom of the Bid Definition Box.

 

You may edit the Seq and the Cnvt, but only for the first meaning. All subsequent meanings must apply to the indicated Cnvt and Seq. If a bid’s description only applys to those situations where an initial pass was made by the bidding side, make sure that the Seq reflects that; e.g. "P-2H", instead of "2H"

If a bid has multiple definitions, pressing the erase button will eliminate the current definition. New definitions may be added using the New Definition button.

Linking to a Convention Card

Top  Previous  Next

 

The Sequence File contains descriptions for all bidding conventions recognized by Bridge Captain. The Convention Card, which is accessed via the CC Editor, indicates which conventions are used by the N-S and E-W teams. The information from these two sources is linked in the following manner.

 

Bridge Captain creates a temporary file to reflect all bidding sequences used by the team. The bids are copied into the temporary file from the Sequence File in the following order. If the same sequence is defined multiple times, the later description will overwrite an earlier description.

 

1. First all [STD] bids are copied.

2. Next, all bids from team's General Approach are copied. For example, the [2/1] bids are next added to this temporary file.

3. If a "Programmed Bidding System" is indicated on the CC, those bids are added next.

4. Bridge Captain next iterates through all other [Cnvt] sections in the Sequence File, adding the sequences from those [Cnvt]'s that were selected on the team's convention card. In some instances, inclusion of a [Cnvt] section is also dependent on the opposition's convention card.

5. Because [Cnvt]'s in the Sequence File are maintained in alphabetical sequence, it is possible for the bids from one convention to inappropriately overwrite bids from another convention. For example, the sequences for Garbage Stayman  would incorrectly be overwritten by some Stayman sequences because [STAY] follows [GARB] in the Sequence File. This problem is solved in Bridge Captain by referencing the CnvtOrder.txt file. The order of Cnvt's mentioned in the CnvtOrder file is determined alphabetically within the order sqeuence. The order sequence number should be a one digit number between 6 and 9.

Top  Previous  Next

 

The allowable format for LTC is "a-b" where "a" and "b" may be integer or decimal numbers; e.g. 4-8, or 6.5-8.5

 

Multiple Meanings

Top  Previous  Next

 

The description of a bidding sequence with multiple meanings has the following structure in the Sequence File:

seq=meaning-1 OR meaning-2 OR ...OR meaning-n

While there is no limit to the number of meanings, the maximum length of the line is 255 characters.

 

If a bid has multiple meanings, you will see the current meaning number and the total number of meanings just to the left of Cnvt in the Bid Definition Box, e.g. "2/3" would indicate the 2nd meaning out of the total of 3 meanings for this description.

 

Click the OR buttons to scroll forward or backward to different meanings.

 

Click the Add Meaning button when you want to add an additional meaning to the bid's description.

 

To remove the current meaning, just click the Erase button

 

When you add or delete meanings from a bid’s description, the counter to the left of Cnvt, will be updated automatically.

 

Add Meaning

Top  Previous  Next

 

Press this button to add a new meaning to the bid's description.  The meaning numbers will automatically be updated.

 

Or Buttons

Top  Previous  Next

 

the "Or >" and "< Or" buttons allow you to see the next, or prior definition for this bid. When a bid has only one definition, these buttons will be disabled.

 

Each time you click an Or button, the definition number will change.

A complete list of the allowable Point Codes can be seen by clicking  "Codes" on the Bid Definition Box’s menu or in the Point Code Table.  For example, you may define a bid’s point range as being "INV".  This would mean the hand’s strength is invitational, showing a total of 23-24 points when added to the minimum points already shown by partner.  This is necessary to accommodate variable point ranges in previous bidding. 

 

A common Point Code to use is "CC" which means that the point range is taken from the pair’s convention card (i.e. from the appropriate .cc file). The definition for a one no trump opening bid in most systems is 1N=PCC B (indicating that the point range is taken from the CC file, and the hand must be balanced).

 

For maximum flexibility, it is generally preferable to use a Point Code, rather than a specific range in points.  Example Point Codes includes: -INV (less than invitational), INV+ (invitational or more), GF, GF+, NF (non-forcing).

Point range

Top  Previous  Next

 

The point range in the bid’s definition may be indicated by either a special Point Code, or by a specific range using one of the four formats allowed, where "n and "m" are integer values:

 

n

for a single point count

n-m

the most common, for a range from "n" to "m", inclusive, e.g. 8-10

0-n

for any point count up to, and including "n"

n+

for a hand with "n" or  more points

 

Points

Top  Previous  Next

 

Usually, but not always, a bid will indicate a certain number of points.  These "points" must be either:

HCP

high card points, based upon the standard 4-3-2-1 system, or

Total points

based upon HCP plus distribution points.

 

Distribution points are calculated by Bridge Captain.  When the bidding commences, a hand’s distribution points equal it’s "length points", where one length point is given for each card in excess of four, in each suit.  For example, AJxxxx KQxxx x Q has 12 HCP, 2 length points in spades with the six card suit, and one length point in hearts, for a 15 Total Points. Bridge Captain adjusts a hand’s total points as the bidding progresses. For example, short points (3 for a void, 2 for a singleton, and 1 for a doubleton)  are added when a hand can support partner's suit.

 

When defining a bid’s points, you may enter a specific number, or range, based upon one of the following formats: n, n+, and n-m.

 

Bridge Captain also allows several Point Codes. The actual points in the current situation are displayed below the box where the code is shown. This number will vary from bidding system to system.  For example, when 1N-2N invites game, the actual number of points shown by this bid will depend upon the points shown by the opening NT bid. The number of points indicated by a "point code" are often dependant on Partner’s Minimum Points (PMP) already revealed in the bidding.

Pressing this button will save the current definition in the Sequence File. You should only press this button if you have made changes to a bid's definition AND you wish to save them.

 

Seq

Top  Previous  Next

 

This is the bidding sequence shown in the Sequence File.  Note that:

1. Opponents' bids are shown within parentheses.

2. Opponents' passes are not shown.

3. An Cnvt which affects the bid must be included in the Cnvt box.

 

For example, a Seq of (@1N)-P-(2D)-3D would define a 3♦ bid after the opponents had opened 1NT and responder had made a 2D bid.  The meaning of this bid is significantly different depending upon whether the 2♦ bid is natural, showing diamonds, or a Jacoby Transfer, showing hearts.  If responder's bid was a Jacoby Transfer, then make sure the bid definition includes (JACH) in the CNVT area.

 

As a short hand to defining multiple bids, you may use "m" for either minor suit and an "M" to represent either Major suit.  In this case, put the suit requirements for the "m" suit under diamonds and the suit requirements for the "M" suit under spades.

 

For example, assume you are defining weak jump shifts for 2 of a major after an opening bid of 1 of a minor. Rather then define all four possibilities individually, you can define "1m-2M".  Under spades you might show 6+/QQ to show that the major suit (either hearts or spades) must be at least 6 long, with at least a queen. Perhaps you might also define that the minor suit should have 3 cards at most, so under diamonds, you would show a length of 0-3.  Upon pressing the Save button in this example, four separate definitions will be placed in the Sequence File: 1C-2H, 1C-2S, 1D-2H, and 1D-2S.

 

Note that this "M" and "m" short-cut feature only applies to sequences entered in the Bid Definition Box.  The Sequence File will never contain these letters to indicate Major or minor suits.

 

Sequence File

Top  Previous  Next

 

The Sequence File is an ASCII text file and is structured like an .INI file:

[Cnvt]

Seq=description

seq=description

.

.

seq=description

[Next Cnvt]

etc.

 

Cnvt refers to a grouping of bids known as a convention or perhaps a system. For example, after [JACS] would follow all the bidding sequences pertaining  to a Jacoby transfer bid to 2S. Multiple Cnvts may be included, with each one separated by an underscore "_" character. Include the convention with parentheses when it refers to the opposition's conventon. For example, the [LEBF_(VNTS>CAPP)] section of the Sequence File covers bidding situations using Lebensohl, Fast denies after the opponents interfer with Cappelletti versus your strong NT.

 

Seq refers to a specific bidding sequence.  The opponents’ bids are contained within parenthesis, e.g. 1C-(1S)-2C.  To save space and make it easier to read, all "pass" calls made by the opponents are omitted, e.g. 1S-3S-4S has omitted the opponent’s passes.  Suits are denoted by the first letter - "C","D","H" or "S", while "N" is used for no trump, "X" for double, and "XX" for redouble. Often a sequence will have the same meaning whether or not the opening bid was made in 3rd or 4th seat.  If Bridge Captain does not find the specific sequence in the Sequence File when the opening bid was made in 3rd or 4th seat, it will next try the sequence without a leading "P-".  For example, if the bidding has gone: pass-(pass)-2C-(pass), Bridge Captain will first look for "P-2C=" in the file.  If that bid is not found, it will then look under  "2C=" for the bid’s meaning.

 

Description is the complete description for the particular bidding sequence. This could be quite simple, such as P15-17 B (indicating 15-17 points and balanced distribution), or it could be very complicated, including multiple meanings. The only limitation is that the ‘seq=description’ character string must not exceed 255 characters.  The complete description for any Seq can be viewed and edited in the Bid Definition Box.

A complete list of codes can been seen by clicking on the Codes menu item of the Bid Definition Box.

/TY=Bid meaning is valid if this is the trump suit (for RKC etc.)

/X=Exclude HCP in this suit from required points

 

Suit Quality Codes

Top  Previous  Next

 

The Suit Quality codes indicate the high cards in the suit, not it's length.

 

The general form is /Qmn where "m" shows the number of cards required out of the top "n" cards.  For example /Q13 indicates that the suit must contain at least 1 of the top 3 honors (Ace, King, or Queen).  You can also use a minus sign to indicate that the suit must weaker than the indicated strength.  For example, /Q-13 indicates that the suit can not hold one of the top three honors.  Both "m" and "n" are limited to 0 through 5.  Another example:  /Q25 indicates the suit must hold 2 of the top 5 honors.

 

The specific form is /QC where "C" shows a specific honor card (A,K,Q,J, or T).  For example, /QK indicates that the suit must contain a King or higher honor card. Alternatively, /Q-Q indicates that the suit must not have a Queen or higher card. 

 

Sometimes the specific form is easier to code, but that approach does not handle multiple card requirements, like /Q23.

 

Suits

Top  Previous 

 

First enter the suit’s required length, if any. The same four formats used for points can be used for suit length. Next you may code any number of optional Suit Codes.  A complete list of all Suit Codes can be found under the Codes menu bar. Each Suit Code must be preceded by a "/" character. 

 

For example, a bid may require spades to be "4-6/QK", which shows that the suit’s length must be between 4-6, inclusive, headed by a King or better.

 

The length qualifier is not required. For example you could enter "/Q23" to indicate that the suit quality must include 2 of the top 3 honors, regardless of length. In this example, the underlying minimum length of the suit is 2 cards. Note that the "/" is required even in those situations where you are not also showing length.
